# 5.0 — Risk Management

## 1. Risk fundamentals

- **Risk = likelihood × impact** (of a threat exploiting a vulnerability against an asset).
- **Threat** (the actor/event) × **vulnerability** (the weakness) × **asset value** → risk.
- **Risk responses (treatment):**
  - **Mitigate/reduce** — apply controls.
  - **Transfer/share** — insurance, outsourcing.
  - **Avoid** — stop the risky activity.
  - **Accept** — formally tolerate (within appetite), with sign-off.
- **Residual risk** = risk left after controls; **inherent risk** = before controls; **risk
  appetite/tolerance** = how much the org will accept.

## 2. Risk assessment methods

- **Qualitative:** likelihood/impact rated (Low/Med/High), **risk matrix/heat map** — fast, subjective.
- **Quantitative (know the formulas):**
  - **SLE** (Single Loss Expectancy) = **Asset Value × EF** (Exposure Factor).
  - **ARO** = Annualised Rate of Occurrence (times/year).
  - **ALE** (Annualised Loss Expectancy) = **SLE × ARO**.
  - Justify a control if its annual cost < the **reduction in ALE** it delivers.
- **Risk register:** living record of risks, owners, scores, treatments, status.
- **Assessment types:** one-time / continuous / ad-hoc; **control assessment**, **risk-based**.

> for finding severity (e.g. directory listing = medium-high), with CVSS as a first-order estimate —

## 3. Risk & security frameworks

| Framework | Purpose |
|-----------|---------|
| **NIST CSF** (Identify, Protect, Detect, Respond, Recover) | Voluntary cyber risk framework — the five functions are high-yield |
| **NIST RMF** (SP 800-37) | Federal risk management process (Categorize→Select→Implement→Assess→Authorize→Monitor) |
| **NIST SP 800-53** | Control catalogue |
| **ISO/IEC 27001 / 27002** | ISMS certification standard / control set |
| **CIS Controls & Benchmarks** | Prioritised controls + hardening baselines |
| **COBIT** | IT governance |
| **PCI DSS** | Card-data security (regulatory/contractual) |
| **SOC 2 / SSAE 18** | Service-org trust reporting |
| **MITRE ATT&CK** | Adversary TTP knowledge base (detection/threat-intel) |

## 4. Business continuity & disaster recovery

- **BIA (Business Impact Analysis):** identifies critical functions and the metrics below.
- **Key metrics (know them precisely):**
  - **RTO** (Recovery Time Objective) — max tolerable *downtime*.
  - **RPO** (Recovery Point Objective) — max tolerable *data loss* (time since last good backup).
  - **MTD** (Maximum Tolerable Downtime) — RTO must be ≤ MTD.
  - **MTBF** (Mean Time Between Failures — reliability), **MTTR** (Mean Time To Repair/Recover),
    **MTTF** (mean time to failure, non-repairable).
- **DR sites:** **hot** (ready, near-instant, costly), **warm** (partial, hours), **cold** (space
  only, days, cheap).
- **Plans:** BCP, DRP, **IRP** (incident response), **COOP**, communications plan, succession.
- **Testing:** **tabletop** (discussion) → **walkthrough** → **simulation** → **parallel** → **full
  interruption** (most realistic, most disruptive).
- **Order of restoration** and **redundancy** (see [domain 3](3.0-architecture-design.md) §5).

## 5. Incident response

- **Lifecycle (NIST SP 800-61):** **Preparation → Identification/Detection → Containment →
  Eradication → Recovery → Lessons Learned**. (Some texts use PICERL.)
- **Containment:** short-term (isolate) vs long-term; **isolation, segmentation, sandboxing**.
- **Roles:** CSIRT/CIRT, incident commander, comms lead; **playbooks/runbooks**.
- **Forensics:** **order of volatility** (RAM/cache → disk → archives), **chain of custody**,
  **write blockers**, imaging, hashing for integrity, legal hold, e-discovery.

## 6. Data privacy, governance & compliance

- **Regulations:** **GDPR** (EU/UK — personal data, breach notification, DPO, data-subject rights),
  HIPAA (US health), PCI DSS (cards), SOX, CCPA. Know GDPR is the big one for UK/EU markets.
- **Data roles:** **data owner**, **data controller** (decides purpose/means), **data processor**
  (acts on controller's behalf), **data custodian/steward** (day-to-day handling), **DPO**.
- **Data lifecycle:** classification → retention → **secure disposal/sanitisation** (wiping,
  degaussing, shredding, crypto-erase).
- **Agreements:** **SLA** (service levels), **MOU/MOA**, **BPA**, **NDA**, **ISA** (interconnection),
  **DPA** (data processing). Vendor/**supply-chain risk management** and third-party assessments.
- **Policies:** AUP, separation of duties, least privilege, onboarding/offboarding, clean desk,
  **security awareness training** (the human control — ties to [mirage](../../mirage)'s phishing focus).

## 7. Security awareness & the human element

User training, **phishing simulations**, role-based training, gamification, recognising social
engineering, reporting culture. People are the most-attacked layer — training is a **preventive +
directive** control.

> Portfolio link: [mirage](../../mirage) studies *why* social engineering works (urgency, authority,
> trust) — the evidence base behind effective awareness programmes.

- Data controller vs processor under GDPR?
