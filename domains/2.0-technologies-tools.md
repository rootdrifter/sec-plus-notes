# 2.0 — Technologies & Tools

## 1. Network security devices

| Device | Function | Key distinction |
|--------|----------|-----------------|
| **Firewall** | Filters traffic by rules | **Stateful** (tracks connections) vs **stateless** (per-packet); **NGFW** adds app-awareness, IPS, TLS inspection |
| **Router / ACLs** | Routing + basic L3/L4 filtering | ACLs = ordered allow/deny, implicit deny at end |
| **Proxy** | Intermediary for requests | **Forward** (clients→internet, caching/filtering) vs **reverse** (internet→servers, LB/WAF/TLS offload) |
| **Load balancer** | Distributes traffic | Modes: round-robin, least-connection; **affinity/sticky sessions**; active/active vs active/passive |
| **IDS / IPS** | Detect / detect+block intrusions | **IDS = detect/alert (out-of-band)**, **IPS = block (inline)**; signature vs anomaly vs behaviour-based; **NIDS vs HIDS** |
| **WAF** | App-layer HTTP filtering | Stops SQLi/XSS etc. (vs network firewall = L3/L4) |
| **VPN concentrator** | Terminates VPN tunnels | Site-to-site vs remote-access; split vs full tunnel |
| **SIEM** | Aggregates + correlates logs | See §4 |
| **DLP** | Prevents data exfiltration | Endpoint/network/cloud; pattern + classification based |
| **NAC** | Controls device network admission | Posture/health checks before granting access; 802.1X |
| **Jump box / bastion** | Hardened admin entry point | Reduces management attack surface |

**Firewall rule essentials:** order matters, top-down first-match, **implicit deny** at the end;
log denies; least privilege per rule.

## 2. Endpoint protection

- **Antivirus / anti-malware:** signature + heuristic + behaviour detection.
- **EDR / XDR:** continuous endpoint telemetry, detection, response, rollback — far beyond classic AV.

## 3. Secure network design tools/protocols

- **Secure vs insecure protocol pairs (know these cold):** SSH↔Telnet, HTTPS↔HTTP, SFTP/FTPS↔FTP,
  SNMPv3↔SNMPv1/2, LDAPS↔LDAP, SMTPS/STARTTLS↔SMTP, DNSSEC (integrity) + **DoH/DoT** (confidentiality)
  ↔ plain DNS.
- **Port knowledge (high-yield):** 22 SSH, 23 Telnet, 25 SMTP, 53 DNS, 80 HTTP, 110 POP3, 143 IMAP,
  443 HTTPS, 389 LDAP / 636 LDAPS, 445 SMB, 3389 RDP, 161/162 SNMP, 3306 MySQL, 1433 MSSQL, 5432
  Postgres, 21 FTP, 69 TFTP, 123 NTP, 88 Kerberos, 514 syslog.

> Portfolio link: [spectre](../../spectre)'s nmap scan found **22/SSH** and **80/HTTP**; the
> remediation included **HTTP→HTTPS** redirect (the insecure→secure protocol upgrade).

## 4. Monitoring, logging, SIEM

- **SIEM** functions: **aggregation, correlation, alerting, dashboards, retention, WORM storage,
  time synchronisation (NTP — essential for correlating logs), automated triggers**.
- **SOAR** adds playbook automation/orchestration on top.
- Log sources: syslog, Windows Event Logs, firewall/IDS/proxy/DNS logs, NetFlow.
- **Key SIEMs to name:** Splunk (SPL), Microsoft Sentinel (KQL), IBM QRadar (AQL), Elastic.
- Concepts: **log aggregation, sensors/collectors, sensitivity/alert tuning** to reduce false
  positives, **correlation rules**.

## 6. Mobile & wireless tools/threats

- **Wireless security:** **WPA3 > WPA2 (AES/CCMP) > WPA/WEP (broken)**; **EAP** methods
  (EAP-TLS strongest, PEAP, EAP-TTLS), **802.1X** with a RADIUS server; **WPS** is weak; disable.
- **Wireless attacks:** evil twin/rogue AP, deauth, jamming, **bluejacking/bluesnarfing**, war
  driving, **IV attack** (WEP), **KRACK** (WPA2).
- **Mobile deployment:** **BYOD / COPE / CYOD / corporate-owned**; **MDM/UEM** for policy,

## 7. Email security tooling (high-yield)

Because email is the #1 attack vector, the three sender-authentication records are heavily tested —
know what each one *does* and how they combine:

| Record | Protects against | Mechanism |
|--------|------------------|-----------|
| **SPF** (Sender Policy Framework) | Sender-IP spoofing | DNS TXT listing the IPs allowed to send for a domain |
| **DKIM** (DomainKeys Identified Mail) | Tampering / forged content | Cryptographic signature in the header, verified via a DNS-published public key |
| **DMARC** | Domain spoofing + tells you what to do | Policy (`none`/`quarantine`/`reject`) built on SPF+DKIM **alignment**, plus reporting (RUA/RUF) |

- **Other email controls:** **Secure Email Gateway (SEG)**, sandboxing of attachments, URL
  rewriting/detonation, **S/MIME / PGP** for end-to-end signing/encryption, **DLP** on outbound mail.

> Portfolio link: DMARC/SPF/DKIM are the technical counter to the social-engineering threat
> [mirage](../../mirage) studies — they cut domain-spoofed phishing before the human ever sees it,
> while mirage addresses what survives the filter and reaches the user.

## 8. Forensic & analysis tooling

- **Disk/memory forensics:** **Autopsy / The Sleuth Kit** (open-source), **EnCase**, **Magnet
  Axiom**, **FTK** (commercial); **Volatility** (memory). **write blockers** to preserve evidence.
- **Imaging & integrity:** `dd`/`dcfldd` for bit-for-bit images; **hashing (MD5/SHA-256)** to prove
  the image is unaltered; maintain **chain of custody** (ties to domain 5 §5).
- **Log/triage:** `grep`, `jq`, Sigma rules, `chainsaw`/`hayabusa` (Windows event triage).
- **Reverse engineering / malware analysis:** Ghidra, `strings`, `file`, sandboxes (Cuckoo, ANY.RUN).

- WPA2 vs WPA3, and why is WEP broken?
- SPF vs DKIM vs DMARC — what does each check, and which one sets the enforcement policy?
